Introduction

Domain and Project Motivation

With a vast amount of trade data available, this project aims at developing a set of visualizations which could help users draw valuable descriptive inferences which in turn, would help the users to understand our visualizations without any sense or feeling of information overload about the imports and exports of different countries from trade data, since this is a bulk dataset. The WITS portal[1] does have certain visualizations that are good in terms of understanding and interpretability. However, there are certain things which we can do through this projects:

  • Due to the size of this dataset, there exist certain visualizations that a set of users may feel stressed to understand. (like high-school students, non-FINTECH background people, etc).
  • Few existing visualizations can be plotted in such a way that their understandability can be improved.

The domain of this project is economics/finance. Potential users are typically the general population. We aim to develop the visualizations that help users compare imports and exports among countries in-detail with better understanding without any prior-knowledge or expertise. In the source website, it consists of some sophisticated visualizations. But through our project The visualizations would help the users answer the following questions:

  • What are the various product categories and what are the indicator values for each category thats imported/exported by a specific country in a specific time frame? (Task-1)
    {action, target} : {Summarize the indicator values of all products for a specific country in a specific time frame, View the proportions of indicator values of various product categories in a given year and timeframe}
  • In a given time frame, what is the trend of net trade for a specific country ? (Task-2)
    {action, target} : {Derive the net indicator value for a specific country over specific time frame, View the trend of net indicator value of a specific country in a specific time frame}
  • In a given time frame, how much trade is done among all the countries for a specific or all product categories? (Task-3)
    {action, target} : {Summarize the trade interactions of a specific country in specific time frame, View how much in terms of aggregated indicator value did a target country trade with partner countries in a specific time frame}
  • For a specific product category, in a specific year,how much (in terms of a specific indicator value) did all the countries import/export? (Task-4)
    {action, target} : {Summarize the proportions of each country’s aggregated indicator value towards a specific product for a specific indicator in a specific timeframe, View the distribution of a product’s net indicator value over all countries}

Data

Data set: We will use the trade data available from WITS (World Integrated Trade Solution website). It contains the trade data related to about 217 countries from the years 1988-2021. Its aggregated by World Integrated Trade Solutions
We will be using the archived version of it from here[4](Trade Stats - Country Summary zip file).

Data Semantics

Data Items: Each row represents trade data (export/import)- for a specific country over the years.

Attribute Description Attribute Type Domain
Indicator Type This represents the nature of the trade, i.e., whether the nature of trade is import, export, developmental fund, or tariff funds. Categorical (Among Import, Export, Development, Tariff) String
Reporter This represents the country of interest, i.e., the specific country which records the trade with the partner country. Categorical (Name of the Country) String
Partner This represents the country with which the reporter has recorded the trade. Categorical String
Product Category This represents the category of goods or commodities which have been traded between the reporter country and the partner country. Comprehensive list of product categories can be found here[2]. Categorical String
Indicator This categorizes the indicator type one level further down. For example, if the indicator type is export, then this attribute also tells if the observation recorded is export in USD $ or just the number of exports etc. Comprehensive list of indicators could be found here[1]. Categorical String
1988-2021 (34 Columns) These attributes record the monetary value of the transactions (in $ USD) or the number of imports/exports (depending on the value of Indicator described in the previous row) that happened between the reporter and partner country. Each year (which is taken as an individual attribute) represents the year in which the transactions have occurred. Nominal Float

Limitations of the Dataset

  • Assuming there exists confidentiality between the two countries regarding their trade relations, there are cases where there exists unknown product categories. This poses a drawback if we want to visualize what/which product was traded between the countries.
  • Missing trade values in some years.

Project Contributions

The questions which we would try to address through the visualizations are:

Task 1: What are the various product categories and in what is the net indicator value of each category for a specific country in a specific time frame?

Achieved using a treemap using Flask and Matplotlib. See "Task 1" in the Implementation section.

Task 2: In a given time frame, what is the trend of net trade for a specific country?

Achieved using a line graphs using Flask and Plotly. See "Task 2" in the Implementation section.

Task 3: In a given time frame, how much trade (import or export) is done by a country with all the other countries?

Achieved using a network graph using Flask, Matplotlib, and Plotly. See "Task 3" in the Implementation section.

Task 4: For a specific product category, in a specific year, how much (in terms of a specific indicator value) did all the countries import/export?

Achieved using a bar graph using Flask and Matplotlib. See "Task 4" in the Implementation section.

Related Work

Description for Wei Chen & Nyuying Wang (2022) Visualizing the changing geographies of international trade, 2000–19, Regional Studies, Regional Science, 9:1, 132-134 [19]: (Comparison of global trade in year 2000 and 2019 in padlet)
The paper visualizes how the total trade of different countries in the world evolved from 2000 to 2019. It shows two different cartograms one for the year 2000 and the other for the year 2019. In each cartogram, the known geographical position of each country is used to locate the country and color encodes the total trade value (Marks are shapes and color channel is used as a magnitude channel to encode net trade value). I found this source as an inspiration because it's in the same domain of our project and also pretty closely related to a few of the tasks which we want to achieve. This is closely related to task-2 of our project. In task-2, we want to see the net indicator value trend for a specific country in a given time period. Here we intended to encode the trend using a line graph where spatial position along the X-axis encodes the year and spatial position along the Y-axis encodes the indicator value. This idiom is chosen because its real estate efficient considering we will only have one country of interest for a chosen time frame. If I were to do vis in paper differently from the authors, I wouldn’t change the idiom because the task in the paper intends to see the trend for all countries and the cartogram seems efficient to show all countries at once . However, the bucketing of colors based on the net trend value chosen by authors didn't seem intuitive to me in some cases. For example, <100 billions of dollars uses darker green and 100-150 billions of dollars uses less luminous green. So, if I were to do it differently, then I would change color encoding with higher net trend values being more luminous than lower net trend values.

Description for Trade flows between top 10 EU largest countries[20]:
The above visualization is a chord diagram which shows the nominal goods trade flow between the top 10 countries in the European Union(EU) from the year 1998 till 2018. Each flow line(marks) represents the trade relation between the two individual countries. The opacity of each flow line indicates the magnitude of trade that occurred between the countries (magnitude channel) and color of each flow line encodes an individual's country’s trade with the other acting as the identity channel. This visualization is a source of inspiration for task-2 and task-3. In task-2, we have analyzed the net trade for a specific country with the other. In task-3, we have analyzed the amount of trade which is done between the reporter country and individual other or all other countries.
For task-2, we have to analyze the trend of the net trade for a specific country in a given time period. For this we have considered only the individual country and its nominal trade value(both imports and exports) and then aggregate of the entire trade for that year. The chord diagram would visualize the trade relation with the other country. Also, the trade value in the chord diagram is not the exact value of trade since it shows the round off figures and is in the scales of 100. This would hinder the user from fetching the accurate information required.With the present idiom, we aim to visualize a line chart which has an x-axis encoding the year and y-axis encoding the trade indicator value.
For task-3, we would analyze the trade value of an individual country with the other countries in a given time frame. With the present idiom of sankey diagram, it is difficult to evaluate the exact trade indicator value for between those countries since it seems to complicate the understanding of the user though the opacity of the flow lines. However, we can transform such visualization into network type data in order to visualize such requirements in an effective way.

Description for “International Collaboration of antimicrobial resistance in the environment”[21].
In this paper the authors have done some analysis on antimicrobial resistance in environment Research. Here they have visualized the collaboration between the multiple countries that have contributed to this research. Here they have used the Network Graph or map to visualize. Along with that they have added the thickness to the connecting lines of the Countries. The thickness indicates the strength of the collaboration between the two countries. The higher the thickness the more strength between the countries. Likewise in Task-3, for the network graph we can add this feature to represent the trade value between the countries. As this can show the strength or relationship between the two countries just at a glance. Hence, we can show the strength of the relation between the partner Country and Reporter Country, the relation between the different partner countries and the reporter country in a particular year with a specific Product Category.

Description for “US Trade Deficit after Trade War Tariffs” [22]:
The above visualization is a Bar graph which shows the USA trade deficit with China in blue bar and USA trade deficit in general with the rest of the world in Grey bar(color: Channel). The position of the lines/bars(Marks) encodes the magnitude of the deficit amounts over time. Higher values(positive) are positioned higher on the chart and lower values (negative) are positioned lower on the chart. I think that this visualization is a great source of inspiration because of multiple reasons but one of the main reasons would be that it is very closely related to the domain of our project and one of the tasks which we want to achieve. The task that we want to achieve and which is closely related to this vis is task 2. For task 2 we are trying to analyze the trend of the net trade for a specific country for which we consider a single country at a time and we consider its trade value which would be export - import and aggregate that entire trade for that individual year. The x-axis encodes the year, while the height of the blue and gray bars represents the deficit amounts in billions of USD. The bar charts make it easy to compare the total vs China amounts for each individual year, but line charts may better highlight the overall trends over time.If I were to do vis in paper differently from the authors, I would likely to use small multiple line charts to also include more countries, rather than just these two categories. The title and axis labels clearly communicate the figure's contents. The use of color to distinguish total vs China is effective given the goal of comparing them. For my purpose of showing trends in more countries, I would use small multiple line charts without color, relying on spatial positioning to encode values for each. This should allow trends across more partners to be compared both horizontally and vertically.

Description for “Countries and Their Research and Development Spending” [23]:
The reference visualization is a Treemap which shows countries and the total USD they spent for Research and Development. The marks are the areas that enclose the country name and spending amount. The size of the areas is a magnitude channel and the larger the size, the greater the amount spent for R&D. The color is an identity channel where countries that are within the same continent are identified by the same color. This vis closely relates to our task 1 and the vis currently chosen for it. In both instances, the color of the treemap indicates what overarching category an area corresponds to. In the reference, it corresponds to the continent that the country falls within where as in our vis it corresponds to the type of good (capital, consumer, intermediate, or raw material) that the entry is apart of. Also, the use of size is the same between both vis’s, meaning a larger size encodes a larger value. For our purposes, I believe that modifying the color of the different areas so the overarching category has more pop (with more vibrant colors) is important but the relation between areas can still be discerned.

Implementation

Task-1

Action,target pair Summarize the indicator values of all the product categories for a specific country in a specific timeframe. View the Indicator values of each product in millions of USD category exported/imported by a specific country in a specific timeframe
Requirements from UI perspective Provided a drop down to select country, import/export type and the year for which indicator values need to seen for
Data Transformation
  1. Based on the country selected from the drop down fetched the summary file for that country in to a dataframe
  2. Preprocessed the data to replace NA values with zeros
  3. For the selected year from the drop down, computed a dictionary that contains keys as product categories and values as aggregated net import/export (based on whats chosen) value in millions of USD for each product category
  4. Encoded the summarized data from previous step in treemap where each product is encoded by an area mark and size of area mark in treemap encodes the net indicator value for that product category

Task-2

Action,target pair Derive the net indicator value for a specific country over specific time frame and View the trend of net indicator value of a specific country in a specific time frame
Requirements from UI perspective Provided a drop down to select the country and two drop downs (from and to) to select start and end of the time period in which the trend for indicator values need to be viewed. Imposed a validation rule to make sure that end of the time period is strictly greater than start of the time frame
Data Transformation
  1. Based on the country values selected in the drop down fetched the summary file of that specific country to a dataframe
  2. Computed a dictionary that contains keys as year values (year>=form drop down's year value and year<=to drop down's year value)and value as net indicator value (total export value in millions of USD for all product categories)-(total import value in millions of USD for all product categories) from the dataframe from above step
  3. Encoded the data computed above using a line chart where spatial position along X -axis encodes the year and spatial position along Y-axis encodes the net indicator value in millions of USD

Task-3

Action,target pair Summarize the trade interactions of a specific country in specific time frame, View how much in terms of aggregated indicator value did a target country trade with partner countries in a specific time frame
Requirements from UI perspective Provided a drop down to select country and year value and import/export
Data transformation
  1. Fetched the summary file of the selected country in the drop down to the dataframe
  2. From the data frame from above step, computed a dictionary that contains keys as the country names with which selected country from dropdown imported(or exported based on import/export value from drop down) with and values as net indicator value of trade in millions of USD (net export/net import depending on whats selected in the dropdown) between the selected country from drop down and the country represented by current key
  3. Encoded the data from above dictionary in a network graph. The nodes in the network graph represent the country of interest (country selected in the drop down) and all the countries represented keys of above dictionary. There is an edge between the country of interest and all the nodes that represent the keys of above dictionary to indicate that import/export has occurred between them. The size of nodes representing the countries from above dictionary encodes the aggregated indicator value (total import/export in millions of USD) of trade between the country of interest and country represented by the node

Task-4

Action,target pair Summarize the aggregated indicator value of each country towards a specific product for a specific indicator in a specific timeframe, View the distribution of a product’s net indicator value over all countries
Requirements from UI perspective Provided a drop down to select product category, year and indicator type (import/export)
Data transformation
    • Created a json file programatically by running a one time script (by reading all the summary files pertaining to all countries)that contains keys as product categories. Values will be the list of transactions pertaining to that product category.Each trasaction is represented by a list of size 6.
      • First element in the list represents reporter country
      • Second element in the list represents partner country
      • Third element in the list represents the year in which trade occurred between reporter and partner country
      • Fourth element in the list represents the type of transaction (import/export) between reporter and partner
      • Fifth element in the list represents the indicator value type i.e, Export in millions of USD/Import in millions of USD
      • Sixth element in the list represents actual indicator value in millions of USD
    • Feteched the list of transactions pertaining to the product category selected from drop down from the above json file to a list
    • By scanning throgh above list of transactions computed a dictionary that contains country names as keys and net import/export value in millions of USD for that country in the year selected in drop down
    • Encoded the data from above dictionary in bar charts where the spatial position along X-axis encodes country and spatial position along Y-axis encodes net indicator value (encoded either in a single bar chart or in multiple bar charts depending on number of countries imported or exported the selected product category in selected year)
